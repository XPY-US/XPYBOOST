## 🔧 How It Works

XPYBOOST only runs standard Windows commands built into every copy of Windows.
Nothing exotic. Nothing dangerous. Here is exactly what runs under the hood:

### 🗑️ Junk Cleaner
```batch
del /q /f /s %TEMP%\*.*                               :: User temp folder
del /q /f /s C:\Windows\Temp\*.*                      :: Windows system temp
del /q /f /s C:\Windows\Prefetch\*.*                  :: Prefetch cache
net stop wuauserv                                      :: Stop Windows Update service
del /q /f /s C:\Windows\SoftwareDistribution\Download :: Update download cache
net start wuauserv                                     :: Restart Windows Update
```

### 💾 RAM Optimizer
```batch
rundll32.exe advapi32.dll,ProcessIdleTasks             :: Clear memory standby list
powershell Disable-MMAgent -MemoryCompression          :: Disable RAM compression
```

### ⚙️ Service Killer
```batch
sc stop "SysMain"   && sc config "SysMain"   start= disabled  :: Superfetch
sc stop "WSearch"   && sc config "WSearch"   start= disabled  :: Search Indexer
sc stop "DiagTrack" && sc config "DiagTrack" start= disabled  :: Telemetry
sc stop "WerSvc"    && sc config "WerSvc"    start= disabled  :: Error Reporting
```

### ⚡ High Performance Mode
```batch
powercfg -setactive 8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c      :: Activate plan
powercfg -setacvalueindex SCHEME_CURRENT ... 100               :: Min CPU = 100%
```

### 🌐 Network Boost
```batch
ipconfig /flushdns
netsh winsock reset
netsh int ip reset
netsh int tcp set global autotuninglevel=normal
netsh int tcp set global chimney=enabled
```

### 🎮 Gaming Mode
```batch
reg add "...SystemProfile"  /v SystemResponsiveness /d 0   :: Max responsiveness
reg add "...Tasks\Games"    /v "GPU Priority"       /d 8   :: Boost GPU priority
reg add "...GameConfigStore"/v GameDVR_Enabled      /d 0   :: Kill Game DVR
```

### 🎨 Visual Stripper
```batch
reg add "...VisualEffects" /v VisualFXSetting       /d 2   :: Best performance mode
reg add "...Personalize"   /v EnableTransparency    /d 0   :: No transparency
reg add "...Advanced"      /v TaskbarAnimations     /d 0   :: No taskbar animations
```

## 🛡️ Safety

### Is XPYBOOST safe to run?

**Yes.** Every single command is a standard Windows operation you could
run manually in an elevated cmd window. The script is under 500 lines —
[read it yourself](XPYBOOST.bat) before running anything.

| Concern | Reality |
|---------|---------|
| **Malware?** | Zero. Every line is readable at [XPYBOOST.bat](XPYBOOST.bat) |
| **Data collection?** | None. The script makes zero network calls |
| **Permanent changes?** | Almost none. Everything re-enables in `services.msc` |
| **Antivirus flags?** | SmartScreen flags ALL unsigned `.bat` files — by design, not danger |
| **Registry edits?** | Visual effect settings only — fully reversible in System Properties |

### 🔄 How to Undo Everything

| What Was Changed | How to Revert |
|-----------------|---------------|
| Disabled services | `Win + R` → `services.msc` → find service → set Startup Type to Automatic |
| Visual effects | `Win + R` → `sysdm.cpl` → Advanced tab → Performance → Settings |
| Power plan | Control Panel → Power Options → select Balanced |
| DNS changes | Network Settings → adapter options → TCP/IPv4 → Obtain DNS automatically |
| Full system rollback | `Win + R` → `rstrui` → select the pre-XPYBOOST restore point |

### 🔐 Automatic Restore Point

XPYBOOST v3.0 creates a **Windows System Restore Point automatically**
before Full Auto Boost makes any changes. If you ever want to roll everything back:

```
Win + R  →  rstrui  →  follow the wizard  →  select XPYBOOST restore point
```

## ❓ FAQ

<details>
<summary><b>Is XPYBOOST actually safe to run?</b></summary>
<br/>

Yes. Every command in the script is a standard Windows batch operation
you could type yourself in an elevated `cmd` window. The full source is
in this repo at [`XPYBOOST.bat`](XPYBOOST.bat) — read it before running.
It is under 500 lines with comments explaining each section.

</details>

<details>
<summary><b>Why does Windows Defender or SmartScreen flag it?</b></summary>
<br/>

Windows SmartScreen flags **all** unsigned `.bat` files by default —
regardless of what the file actually contains. It is a code-signing
trust system, not a malware scanner. Click **More info** → **Run anyway**.
The entire source is public here for you to audit before trusting it.

</details>

<details>
<summary><b>Can I undo all the changes XPYBOOST makes?</b></summary>
<br/>

Yes. Almost everything is reversible:

- **Services** → `Win + R` → `services.msc` → set Startup Type back to Automatic
- **Visual effects** → System Properties → Advanced → Performance → Settings
- **Power plan** → Control Panel → Power Options → select Balanced
- **Full rollback** → `Win + R` → `rstrui` → pick the XPYBOOST restore point

Check `XPYBOOST_Log.txt` on your Desktop to see exactly what was changed.

</details>

<details>
<summary><b>Does it work on Windows 10?</b></summary>
<br/>

Yes. XPYBOOST is tested on both Windows 10 and Windows 11.
Every command is compatible with both versions. A few visual tweaks
may look slightly different between versions but function identically.

</details>

<details>
<summary><b>How often should I run it?</b></summary>
<br/>

| Tool | Frequency |
|------|-----------|
| Junk Cleaner + RAM Optimizer | Weekly |
| Service Killer + Power Plan | Once — changes persist until you revert |
| Network Boost | Whenever your connection feels slow |
| Health Score | Anytime you want to check your PC state |

</details>

<details>
<summary><b>Why not just use CCleaner?</b></summary>
<br/>

CCleaner suffered a [verified malware distribution incident in 2017](https://www.ccleaner.com/news/blog/2017/9/18/security-notification-for-ccleaner-v5336162-and-ccleaner-cloud-v1073191-for-32-bit-windows-users),
installs its own persistent background startup service, nags you to upgrade
on every launch and does not touch service management or power plans at all.

XPYBOOST does everything CCleaner does and more — with zero bloat,
zero cost and every line of code publicly readable before you run it.

</details>

<details>
<summary><b>Will this void my laptop warranty?</b></summary>
<br/>

No. Every operation in XPYBOOST targets Windows software settings only.
Nothing at the hardware, firmware or BIOS level is touched at any point.

</details>

<details>
<summary><b>My antivirus deleted the file. What do I do?</b></summary>
<br/>

Add an exclusion for `XPYBOOST.bat` in your antivirus settings before
downloading, or temporarily disable real-time protection during the download.
The file contains zero executable binary code — only plain text Windows
batch commands that you can read in any text editor.

</details>

## 🤝 Contributing

All contributions are welcome — bug reports, new optimizations, UI improvements
or documentation fixes. Here is how to get involved:

### 🐛 Reporting a Bug

```
1. Open Issues → New Issue
2. Describe what happened vs what you expected
3. Include your Windows version and hardware specs
4. Attach XPYBOOST_Log.txt from your Desktop if relevant
```

### 💡 Suggesting a Feature

```
1. Open an Issue with the "enhancement" label
2. Describe the optimization and why it helps low-end laptops
3. Include the Windows commands if you already know them
```

### 🔀 Submitting a Pull Request

```bash
# Fork the repo on GitHub, then clone your fork
git clone https://github.com/YOUR-USERNAME/LaptopBoost.git

# Create a descriptive feature branch
git checkout -b feature/your-optimization-name

# Make your changes and test on Windows 10 AND Windows 11 as Administrator

# Commit with a clear conventional message
git commit -m "feat: add [optimization name] to gaming mode"

# Push to your fork and open a PR against main
git push origin feature/your-optimization-name
```

### 📐 Contribution Guidelines

- Test every command on **both Windows 10 and Windows 11** as Administrator before submitting
- All changes **must be reversible** by the end user without needing XPYBOOST
- Keep the gaming aesthetic and consistent spacing in all `.bat` UI sections
- Add a comment to every new command explaining what it does and why it helps
- Update `CHANGELOG.md` with your changes under an `Unreleased` section
